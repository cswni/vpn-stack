Pi-hole

Network-wide DNS sinkhole. Blocks ads and trackers at the DNS level for all connected devices.

Caddy

Automatic HTTPS reverse proxy. Obtains and renews Let's Encrypt certificates with zero config.

⚡ Quick Start

  1. Clone the repository 
    git clone https://github.com/cswni/vpn-stack.git
cd vpn-stack
  2. Run the setup script 
    sudo bash setup.sh

    The script will prompt for your VPS IP, passwords, and timezone.

  3. Point your DNS records

    Create two A records pointing to your VPS IP:

    vpn.yourdomain.com  →  YOUR_VPS_IP
dns.yourdomain.com  →  YOUR_VPS_IP
  4. Access your panels

    🔐 VPN Panel: https://vpn.yourdomain.com

    🚫 DNS Panel: https://dns.yourdomain.com/admin

💡 The setup script handles Docker installation, port 53 liberation, image pulling, and stack deployment automatically.

🔧 Manual Setup

1️⃣ Create environment file

cp .env.example .env
nano .env

2️⃣ Edit Caddyfile with your domains

nano caddy/Caddyfile

3️⃣ Free port 53 and deploy

sudo systemctl stop systemd-resolved
sudo systemctl disable systemd-resolved
echo "nameserver 1.1.1.1" | sudo tee /etc/resolv.conf

sudo docker compose up -d

🌐 Network Layout

📡 172.20.0.0/24 (vpn_net)
  ├── .2  🔐 wg-easy
  └── .3  🚫 pihole

WireGuard clients receive IPs in the 10.8.0.0/24 range. DNS queries are automatically routed to Pi-hole.

📄 Caddyfile

vpn.yourdomain.com {
    reverse_proxy wg-easy:51821
}

dns.yourdomain.com {
    reverse_proxy pihole:80
}

💡 Caddy automatically obtains and renews TLS certificates from Let's Encrypt. No extra configuration needed.

📱 Adding VPN Clients

  1. Open the VPN panel

    Navigate to https://vpn.yourdomain.com and log in.

  2. Create a new client

    Click + New Client and give it a name (e.g., "iPhone", "Laptop").

  3. Connect your device

    📷 Mobile: Scan the QR code with the WireGuard app.

    💾 Desktop: Download the .conf file and import it.

🔍 Troubleshooting

🚫 Pi-hole not blocking ads

Verify the listening mode is set to accept queries from VPN clients:

docker exec pihole grep listeningMode /etc/pihole/pihole.toml
# Should show: listeningMode = "ALL"

If it shows LOCAL, recreate Pi-hole:

docker compose down pihole
rm -f pihole/etc-pihole/pihole.toml
docker compose up -d pihole
🌐 DNS not resolving through VPN

Check that Pi-hole can reach upstream DNS:

docker exec pihole dig @127.0.0.1 google.com +short

If it times out, verify upstream configuration:

docker exec pihole grep -A4 "upstreams" /etc/pihole/pihole.toml
🔒 Caddy not issuing TLS certificates

Ensure DNS A records point to the VPS and ports 80/443 are open:

dig vpn.yourdomain.com +short
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
📋 Viewing service logs 
docker logs wg-easy     # 🔐 VPN logs
docker logs pihole      # 🚫 DNS logs
docker logs caddy       # 🌐 Proxy logs
🔄 Restarting the stack 
# Restart everything
docker compose restart

# Full rebuild
docker compose down
docker compose up -d

⚠️ If you need to reset Pi-hole completely, delete pihole/etc-pihole/pihole.toml before restarting. The environment variables in docker-compose.yml will regenerate it with the correct settings.
